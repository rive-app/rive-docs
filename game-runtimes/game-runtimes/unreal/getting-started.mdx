This guide walks you through installing the Rive Unreal plugin, importing a `.riv` file, and rendering your first **Artboard**.

<Info>
Supported on Unreal Engine 5.7.3 and above.
</Info>

## 1. Install the Plugin

<Tabs>
  <Tab title="Fab (Recommended)">
    <Steps>
      <Step>
        Open the **Epic Games Launcher** or Unreal Editor.
      </Step>
      <Step>
        Open **Fab**.
      </Step>
      <Step>
        Search for **Rive** or go directly to  
        https://www.fab.com/portal/listings/4a79a0af-7614-42bd-8857-5375c2668706/
      </Step>
      <Step>
        Add the plugin to your engine or project.
      </Step>
      <Step>
        Restart Unreal Engine if prompted.
      </Step>
    </Steps>
  </Tab>

  <Tab title="GitHub">
    <Steps>
      <Step>
        Clone the repository:

        ```bash
        git clone https://github.com/rive-app/rive-unreal.git
        ```
      </Step>
      <Step>
        Copy the **Rive** plugin folder into your project's `Plugins/` directory.
      </Step>
      <Step>
        Regenerate project files and build.
      </Step>
    </Steps>
  </Tab>
</Tabs>

## 2. Enable the Plugin

1. Open **Edit → Plugins**.
2. Locate **Rive** under Runtime.
3. Enable the plugin.
4. Restart the editor if required.

## 3. Import a Rive File

1. Open the **Content Browser**.
2. Drag and drop a `.riv` file into your project.

Unreal creates a **Rive File** asset.

A **Rive File** contains:

- One or more **Artboards**
- Optional **State Machines**
- Optional **ViewModels**

<Info>
A **Rive File** is an Unreal asset. It does not render directly.
</Info>

## 4. Create and Render an Artboard

Rendering requires creating a runtime **Artboard** from a **Rive File**.

The high-level flow is:

1. Load a **Rive File**
2. Create an **Artboard**
3. Bind a **ViewModel** (if used)
4. Tick the **Artboard**
5. Render the **Artboard**

<Warning>
**Artboards do not tick or render automatically.**  
You must advance them explicitly each frame.
</Warning>

## 5. Drive Logic with ViewModels

Interaction is performed through **ViewModels**.

You can:

- Set **ViewModel** properties from Unreal
- Observe **ViewModel** property changes
- Use **Trigger Properties** for one-shot signals

Legacy direct inputs and event callbacks are deprecated.
