## Intro

In this tutorial, we'll look at an advanced file and look into some of the techniques used. While we won't build the file step by step, we will examine the design, rig, animations, and state machine. For a more in-depth look, dive into the source file and look around.

### 2. Rim lighting

Rim lighting helps your character stand out in the composition.

![Image](https://1159711764-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-M3EXlibk6bj2FzPQW-9%2F-Mgvz3pORBTT0EiK5cuj%2F-Mgw-GXfl2YRpb-jHkqu%2FScreen%20Shot%202021-08-12%20at%202.14.48%20PM.png?alt=media&token=43f90d8e-a87f-4b71-af2f-731ed948579b)

Duplicate shapes and change their color to easily light one or multiple sides.

![Image](https://1159711764-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-M3EXlibk6bj2FzPQW-9%2F-MgwAf2ih4bzt_Tv-8GN%2F-MgwBy0b9EIiPPtMho3n%2F2021-08-12%2015.05.23.gif?alt=media&token=ae9f2482-c8ac-46f4-bd38-823f0f2536d7)

Use gradient strokes on your shapes. As the light falls off, turn down the opacity or select a darker color.

![Image](https://1159711764-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-M3EXlibk6bj2FzPQW-9%2F-MgwDqBbi11nsZEZk3TH%2F-MgwEeSfsgLvtcZ4R6qd%2F2021-08-12%2015.16.37.gif?alt=media&token=214fc1ac-1fae-4220-a922-09cf54899a15)

## Rigging

Start with a basic skeleton.

![Image](https://1159711764-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-M3EXlibk6bj2FzPQW-9%2F-MhEZQbcwLER1DlgVqRD%2F-MhEayekV8Yy2xFJSxf-%2F2021-08-16%2009.32.59.gif?alt=media&token=7a7d384c-ab55-48c3-b69a-71af5363da1e)

Add in additional bones for accessories.

![Image](https://1159711764-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-M3EXlibk6bj2FzPQW-9%2F-MhEZQbcwLER1DlgVqRD%2F-MhEfAxiuNGidtC_E51S%2F2021-08-16%2009.52.06.gif?alt=media&token=e98c5253-2acb-4dde-a4b1-22480a9c14df)

Ensure the bones have the proper hierarchical relationships.

![Image](https://1159711764-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-M3EXlibk6bj2FzPQW-9%2F-MhFFxOoQKuLdNwLrsuu%2F-MhFIC6c_gIWmV4y_4SK%2F2021-08-16%2012.41.45.gif?alt=media&token=d0cb7138-7c90-4695-b2d3-af302de7960c)

Set up [IK constraints](https://help.rive.app/editor/constraints/ik-constraint) to make controlling the legs easier.

![Image](https://1159711764-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-M3EXlibk6bj2FzPQW-9%2F-MhFJg5i18m0p8Gpv-hL%2F-MhFJgSmwbGjR_Urv7HR%2F2021-08-16%2012.51.42.gif?alt=media&token=0316c32a-3db8-46b5-b442-4f307600b237)

For the best performance, [bind and weight](https://help.rive.app/editor/manipulating-shapes/bones#2-binding) only the shapes that will be influenced by two or more bones.

![Image](https://1159711764-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-M3EXlibk6bj2FzPQW-9%2F-MhFK649iTpKqBe6lvYp%2F-MhFNR6ABWKFHjwDgAyl%2F2021-08-16%2013.09.57.gif?alt=media&token=0c1e0894-6e01-49cd-8adb-1eef3561624d)

Use [parent-child relationships](https://help.rive.app/editor/manipulating-shapes/bones#1-hierarchical-relationships) for shapes or groups that only use a single bone.

![Image](https://1159711764-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-M3EXlibk6bj2FzPQW-9%2F-MhFK649iTpKqBe6lvYp%2F-MhFOSNCeGOIXTa_XOyK%2F2021-08-16%2013.14.20.gif?alt=media&token=70d381ea-c8bb-4457-89d9-a70e5998aab0)

Ensure all objects are nested under a Root group before animating. This group will allow you to control the transform properties of the entire composition, even if you've already added some animations.

![Image](https://1159711764-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-M3EXlibk6bj2FzPQW-9%2F-MhFOh0IL8UwoRv2PKhX%2F-MhFQPOtBz4EFDhXr9ox%2F2021-08-16%2013.20.51.gif?alt=media&token=9b546aff-0926-4dc3-9c29-5dd6c4f5d48d)

## Animation

Animate the idle first. Let the control bone do most of the work.

![Image](https://1159711764-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-M3EXlibk6bj2FzPQW-9%2F-MhFTUCSCwvaKuH84-MW%2F-MhFVJo5v2QXE-swmP0b%2F2021-08-16%2013.45.18.gif?alt=media&token=8f28227e-dafb-4d20-af04-f4443514a233)

Add secondary motion to make the animation more interesting.

![Image](https://1159711764-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-M3EXlibk6bj2FzPQW-9%2F-MhFTUCSCwvaKuH84-MW%2F-MhFWCVSiwmndvzB238K%2F2021-08-16%2013.45.55.gif?alt=media&token=cce5fe70-0e60-4d8d-8971-2fc1a55fd7b3)

If you are planning on using the animation in the [state machine](https://help.rive.app/editor/state-machine), be sure to key all objects (bones, shapes, paths, ect...) in their current position. This will ensure the character can return to the proper pose when switching between states.

![Image](https://1159711764-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-M3EXlibk6bj2FzPQW-9%2F-MhFTUCSCwvaKuH84-MW%2F-MhFYPKOLZzMdrLmY7s2%2F2021-08-16%2013.57.27.gif?alt=media&token=3acce67e-f069-4c74-b211-51a7ae2da546)

Use the bones to help you achieve the proper poses, then edit the timing.

![Image](https://1159711764-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-M3EXlibk6bj2FzPQW-9%2F-MhFTUCSCwvaKuH84-MW%2F-MhFdUgbeOUIf5fzcbNH%2F2021-08-16%2014.20.15.gif?alt=media&token=d5e8cf39-c1f1-454b-aa1a-6828f0cecd07)

Once you are happy with the main poses and timing, add in secondary motion.

![Image](https://1159711764-files.gitbook.io/~/files/v0/b/gitbook-legacy-files/o/assets%2F-M3EXlibk6bj2FzPQW-9%2F-MhFTUCSCwvaKuH84-MW%2F-MhFhaSyPfLluXzRWQ1_%2F2021-08-16%2014.38.23.gif?alt=media&token=b899f271-7068-4655-ab17-ffc1c883c686)
